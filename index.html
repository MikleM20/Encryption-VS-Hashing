What is Encryption?

Encryption is the TWO-WAYprocess of scambling information in a way such that it can be unscambled/decrypted by someone else using a key.

To encrypt data people use a cipher, which is pretty much another way of saying algorithm; a list of steps and rules to follow. Knowing how this cipher works is usually the key required to decrypt a piece of encrypted data.

How does it Work?

As said about a cipher/algorithm is used to encrypt and decrypt data. Many types of algorithms exist to encrypt data. Here are some easy to visualize examples.
  • Shift Cipher: Probably the simplest example, this cipher simply shifts the alphabet by a fixed amount.
  • Substitution Cipher: Using this cipher, plaintext characters are replaced systematically, a faily simple example of this type of cipher is ROT13, which places the english alphabet of 26 characters into 2 arrays of 13, a-m and n-z. Using these 2 arrays, you replace a character with the index from one array with the character at the same index in the opposite array. An example would be replacing all 'A's with 'N's and vice versa as they are both at the index 0 of the 2 opposing arrays.
  • Transposition Cipher: This algorithm uses a set of rules, which serve as the key to encrypt data. One example of this type of cipher is the Rail Fence Cipher, which places the plaintext in a graph with n "tracks"/"rails" in a zig-zagonal pattern then using creating the output from what is in the "rails".

Example Of When It Would be Helpful

Imagine you are sending or recieving data online, if that data is valuable you would most likely want to encrypt the data so that the information you are sending is more secure. However, since you must also send the recipient the key, if the attacker gets access to both the data and key, it is possible that they can decrypt the data.
